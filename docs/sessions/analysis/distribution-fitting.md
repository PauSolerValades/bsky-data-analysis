# Distribution fitting results — Sessions

**Date:** 2026-05-23  
**Scripts:** `sessions/analysis/fit_distributions.R` + `sessions/analysis/plot_parameters.py`  
**Input:** `pau_db.sessions_all` (47.4M sessions, 1.57M users) + `pau_db.sessions_engagement` (19.6M sessions, 1.30M users)  
**Method:** Per-user MLE fitting of power-law, exponential, Weibull, lognormal, and gamma distributions to session durations and inter-session gaps. Power-law uses KS-based xmin selection (Clauset et al. 2009). Best-fit selected via Vuong's LLR test, falling back to minimum AIC.

- `sessions/analysis/results_new/params/params_duration.csv` (234 MB)

## Interpretation

1. **Power-law dominates both tables.** 50–63% of users' best-fit distribution is power-law for both durations and gaps. The median α for durations is ~2.4 (infinite variance), for gaps it's 2.0–3.1 (ranging from infinite to finite variance).

2. **Lognormal and Weibull are the only credible alternatives.** Exponential fits fewer than 13% of users (it cannot handle the heavy tail). Gamma fits fewer than 2%. The runner-up for gaps is lognormal (25.9% for all, 13.2% for engagement).

3. **Scale differences are 10–60× between tables.** Every distribution's central tendency parameter (xmin, Weibull λ, lognormal exp(μ), gamma mean) is an order of magnitude or more larger for sessions_engagement. This is the "likes fill the timeline" effect: with likes, sessions last minutes and gaps last hours; without likes, sessions last hours and gaps last days.

4. **Weibull shape reveals behavioral patterns.**
   - **Durations**: both tables have median k ≈ 1.6–1.8 (mildly increasing hazard — sessions tend to end). 21–24% have k < 1 (the longer you browse, the more engaged you get).
   - **Gaps**: sessions_all gap k = 1.08 (near-exponential, 46% k<1 — no natural return time) vs sessions_engagement gap k = 2.65 (strong deadline effect, only 14% k<1 — you *will* come back to post/reply within a predictable window).

5. **Gap regularities differ.** sessions_all gaps are more variable (lognormal σ = 0.96) than engagement gaps (σ = 0.39). Browsing is erratic; content creation follows a rhythm.

6. **The xmin values are large.** Median power-law xmin for gaps is 5,806 s (all) and 36,896 s (engagement). When fitting these distributions for simulation, the body of the distribution (below xmin) needs separate treatment — the power-law only describes the tail.

## Simulation procedure — sampling a synthetic user

Given the fitted per-user parameter distributions, a synthetic user is generated
by the following two-stage procedure:

1. **Sample the best-fit distribution family** (categorical draw from the
   breakdown table below).
2. **Sample the parameters** for that family from the empirical percentile
   distributions (tables in previous sections). Parameters with non-zero
   correlation are sampled jointly; independent parameters are sampled
   marginally.

### Parameter correlation summary

| Distribution | Params | ρ (all) | ρ (engagement) | Sampling |
|---|---:|---:|:---:|
| Power-law | α, xmin | 0.02 | 0.01 | **independent** |
| Weibull (dur) | log shape, log scale | 0.19 | 0.20 | independent |
| Weibull (gap) | log shape, log scale | 0.55 | 0.35 | **joint** |
| Lognormal (dur) | μ, σ | -0.32 | -0.35 | **joint** |
| Lognormal (gap) | μ, σ | -0.64 | -0.57 | **joint** |

Power-law α and xmin are essentially uncorrelated — sample each from its
marginal percentile distribution. Weibull gap and all lognormal pairs show
moderate-to-strong correlation — use the empirical joint distribution
(e.g., sample a user uniformly and take their parameter pair).

### Zero-duration sessions

33.2% of sessions_all and 22.7% of sessions_engagement have duration = 0.
For simulation, after sampling duration from the fitted distribution, flip a
biased coin:
- sessions_all: P(duration → 0) = 0.332
- sessions_engagement: P(duration → 0) = 0.227

Zero-duration sessions represent co-occurring actions at the same microsecond.
They should still be counted as sessions (they contain real actions), but
their duration is 0.

---

### Python pseudocode

```python
import numpy as np

def sample_duration(rng, table="sessions_all"):
    """Sample a session duration for a synthetic user."""
    if table == "sessions_all":
        families = ["powerlaw", "exponential", "weibull", "lognormal", "gamma"]
        probs = [0.530, 0.126, 0.095, 0.090, 0.021]
    else:  # sessions_engagement
        families = ["powerlaw", "weibull", "lognormal", "exponential", "gamma"]
        probs = [0.557, 0.178, 0.127, 0.026, 0.005]

    fam = rng.choice(families, p=probs)

    if fam == "powerlaw":
        # Sample α and xmin independently from their empirical percentiles
        alpha = sample_from_percentiles(rng, alpha_percentiles[table])
        xmin = sample_from_percentiles(rng, xmin_percentiles[table])
        return (rng.pareto(alpha - 1) + 1) * xmin  # shifted Pareto

    elif fam == "weibull":
        # Durations: sample independently (ρ ≈ 0.2)
        k = sample_from_percentiles(rng, weibull_k_percentiles[table])
        lam = sample_from_percentiles(rng, weibull_lam_percentiles[table])
        return lam * rng.weibull(k)

    elif fam == "lognormal":
        # Joint sample: pick a random user, take their (μ, σ) pair
        mu, sigma = sample_joint_lognormal_params(rng, table, "duration")
        return rng.lognormal(mu, sigma)

    elif fam == "exponential":
        rate = sample_from_percentiles(rng, exp_rate_percentiles[table])
        return rng.exponential(1 / rate)

    elif fam == "gamma":
        k = sample_from_percentiles(rng, gamma_k_percentiles[table])
        beta = sample_from_percentiles(rng, gamma_rate_percentiles[table])
        return rng.gamma(k, 1 / beta)


def sample_gap(rng, table="sessions_all"):
    """Sample an inter-session gap for a synthetic user."""
    if table == "sessions_all":
        families = ["powerlaw", "lognormal", "weibull"]
        probs = [0.506, 0.259, 0.223]
    else:
        families = ["powerlaw", "weibull", "lognormal"]
        probs = [0.626, 0.135, 0.132]

    fam = rng.choice(families, p=probs)

    if fam == "powerlaw":
        alpha = sample_from_percentiles(rng, gap_alpha_percentiles[table])
        xmin = sample_from_percentiles(rng, gap_xmin_percentiles[table])
        return (rng.pareto(alpha - 1) + 1) * xmin

    elif fam == "weibull":
        # Joint sample (ρ = 0.35–0.55)
        k, lam = sample_joint_weibull_params(rng, table, "gap")
        return lam * rng.weibull(k)

    elif fam == "lognormal":
        # Joint sample (ρ = -0.57 to -0.64)
        mu, sigma = sample_joint_lognormal_params(rng, table, "gap")
        return rng.lognormal(mu, sigma)
```

The `sample_from_percentiles` function does inverse-transform sampling:
given the P1–P99 values for a parameter, interpolate between them to
generate a continuous random variate. The joint sampling functions pick
a random user from the empirical dataset and return their parameter pair,
preserving the observed correlation structure.
